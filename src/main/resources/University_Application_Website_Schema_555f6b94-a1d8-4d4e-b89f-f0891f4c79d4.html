University Application Website — Database Schema

Database schema design for a university application website where students can:

• Browse and apply to universities by country

• Raise enquiries for platform services (Visa Assistance, Scholarship Guidance, University Counselling, etc.)

• Submit reviews and general feedback

Tables

1. countries

ColumnTypeConstraintsDescription
country_idINTPK AUTO_INCREMENTUnique country identifier
nameVARCHAR(100)NOT NULLCountry name
codeCHAR(2)NOT NULL, UNIQUEISO country code (e.g. IN, UK)

2. universities

ColumnTypeConstraintsDescription
university_idINTPK AUTO_INCREMENTUnique university identifier
nameVARCHAR(200)NOT NULLUniversity name
locationVARCHAR(200)NOT NULLCity/region
image_urlTEXTBanner/logo image URL
descriptionTEXTUniversity description
country_idINTFK → countriesCountry the university belongs to
created_atTIMESTAMPDEFAULT NOW()Record creation time

3. services

Platform-level service catalogue. Examples: Visa Assistance, Scholarship Guidance, University Counselling. Services exist independently — students raise enquiries against them via the enquiries table.

ColumnTypeConstraintsDescription
service_idINTPK AUTO_INCREMENTUnique service identifier
nameVARCHAR(100)NOT NULLService name (e.g. Visa Assistance)
contentTEXTNOT NULLService description

4. students

ColumnTypeConstraintsDescription
student_idINTPK AUTO_INCREMENTUnique student identifier
nameVARCHAR(100)NOT NULLStudent full name
emailVARCHAR(150)NOT NULL, UNIQUEStudent email address
phoneVARCHAR(20)Contact phone number
created_atTIMESTAMPDEFAULT NOW()Record creation time

5. enquiries NEW

Junction table linking students to services. A student can raise enquiries for multiple services; a service can receive enquiries from many students.

ColumnTypeConstraintsDescription
enquiry_idINTPK AUTO_INCREMENTUnique enquiry identifier
student_idINTFK → studentsStudent raising the enquiry
service_idINTFK → servicesService being enquired about
messageTEXTNOT NULLStudent's specific question or request
statusENUMopen / in_progress / resolvedEnquiry handling status
created_atTIMESTAMPDEFAULT NOW()Enquiry submission time

6. applications

ColumnTypeConstraintsDescription
application_idINTPK AUTO_INCREMENTUnique application identifier
student_idINTFK → studentsApplying student
university_idINTFK → universitiesTarget university
statusENUMpending / accepted / rejectedApplication status
applied_atTIMESTAMPDEFAULT NOW()Application submission time

7. reviews

ColumnTypeConstraintsDescription
review_idINTPK AUTO_INCREMENTUnique review identifier
contentTEXTNOT NULLReview text
ratingINTCHECK (1–5)Numeric rating
student_idINTFK → studentsReviewer
university_idINTFK → universitiesReviewed university
created_atTIMESTAMPDEFAULT NOW()Review submission time

8. feedback

ColumnTypeConstraintsDescription
feedback_idINTPK AUTO_INCREMENTUnique feedback identifier
contentTEXTNOT NULLFeedback text
student_idINTFK → studentsStudent who submitted feedback
created_atTIMESTAMPDEFAULT NOW()Submission time

Table Relationships

RelationshipTypeVia
country → universitiesOne-to-Manyuniversities.country_id
student → applicationsOne-to-Manyapplications.student_id
university → applicationsOne-to-Manyapplications.university_id
student ↔ servicesMany-to-Manyenquiries (junction table)
student → reviewsOne-to-Manyreviews.student_id
university → reviewsOne-to-Manyreviews.university_id
student → feedbackOne-to-Manyfeedback.student_id

API Endpoints Summary

#MethodEndpointPurpose
1GET/page?country_id={id}Page load — universities, services, reviews
2GET/universities/{id}University details
3POST/applicationsApply to a university
4POST/enquiriesSubmit a service enquiry (student + service + message)
5POST/reviewsSubmit a university review
6POST/feedbackSubmit general feedback
7GET/universities/{id}/reviewsReviews for one university
